HOW TO USE
================================================================================

ADMIN - CREATE PROJECT WITH MEMBERS
1. Go to Projects → Create New Project
2. Fill Step 1: Title & Description
3. Step 2: Select Team Members (checkboxes)
4. Step 3: Upload cover (optional)
5. Submit → Members get notified

ADMIN - EDIT PROJECT MEMBERS
1. Go to Projects → Edit existing project
2. In Team Members dropdown:
   - Check new members to add
   - Uncheck existing to remove
3. Save → Only new/removed get notifications

EMPLOYEE - VIEW ASSIGNED PROJECTS
1. Go to Dashboard
2. See "My Projects" section
3. Only shows projects where you're assigned
4. Click to view project details

ADMIN - FILTER TEAM BY PROJECT
1. Go to Team Management
2. Use "Filter by Project" dropdown:
   - "All Projects" = all workspace members
   - Specific project = only assigned members

================================================================================
NOTIFICATIONS SYSTEM
================================================================================

When project created with members:
  Type: "project_created"
  Message: "You have been assigned to project {name}"
  Recipients: All assigned members

When member added to project:
  Type: "user_added"
  Message: "You have been added to project {name}"
  Recipients: Only new member(s)

When member removed from project:
  Type: "user_removed"
  Message: "You have been removed from project {name}"
  Recipients: Only removed member(s)

When project updated:
  Type: "project_updated"
  Message: "{name} was updated" or "marked as completed"
  Recipients: All team members (except updater)

================================================================================
API ENDPOINTS
================================================================================

POST /api/projects
  Body: { title, description, status, priority, deadline, teamMembers: [ids] }
  Returns: Created project with populated teamMembers

GET /api/projects
  Non-admin: Returns projects where teamMembers contains userId
  Admin: Returns all workspace projects

GET /api/projects/:id
  Returns: Project with populated teamMembers
  Access: Assigned member or admin only

PATCH /api/projects/:id
  Body: { title, description, status, priority, deadline, teamMembers: [ids] }
  Returns: Updated project
  Notifications: Sent for new/removed members

GET /api/projects/:id/members
  Returns: { members: [populated teamMembers] }
  Access: Assigned member or admin only
